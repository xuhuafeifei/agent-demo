# 流式消息时序说明文档

> 如何实现 thinking → toolcall → text 交错显示

---

## 核心原理

**所有消息（thinking、toolcall、assistant）通过时间戳排序实现交错显示**

---

## 后端 Event 格式

### 1. streamStart - 流式开始

```json
{
  "type": "streamStart",
  "timestamp": 1712000000000
}
```

### 2. agent_thought_chunk - 思考内容

```json
{
  "type": "agent_thought_chunk",
  "content": "让我想想这个问题...",
  "timestamp": 1712000000000
}
```

**关键**: Thinking 的时间戳应该比下一个 assistant 消息**早 1ms**

### 3. tool_call - 工具调用开始

```json
{
  "type": "tool_call",
  "toolCallId": "tc_001",
  "toolName": "read_file",
  "title": "正在读取文件",
  "content": "/path/to/file.txt",
  "status": "running",
  "timestamp": 1712000001000
}
```

### 4. tool_call_update - 工具调用更新

```json
{
  "type": "tool_call_update",
  "toolCallId": "tc_001",
  "status": "completed",
  "detail": "完成 (120ms)",
  "timestamp": 1712000001500
}
```

### 5. agent_message_chunk - AI 回复（toolcall 后继续）

```json
{
  "type": "agent_message_chunk",
  "content": "我已经读取了文件内容...",
  "timestamp": 1712000002000
}
```

**关键**: 这个 timestamp 比 toolcall 晚，会创建**新的 assistant 消息段**

### 6. streamEnd - 流式结束

```json
{
  "type": "streamEnd",
  "timestamp": 1712000003000
}
```

## 完整时序示例

### 场景：thinking → toolcall → text 交错

```
时间轴 →

T0 (1712000000000)
  ↓
  后端发送：streamStart
  前端：创建 assistant 占位消息（timestamp: T0）

T1 (1712000000000)
  ↓
  后端发送：agent_thought_chunk { content: "让我想想..." }
  前端：创建 thinking 消息（timestamp: T0 - 1 = 1711999999999）
       → 排序后在 assistant 前面

T2 (1712000001000)
  ↓
  后端发送：tool_call { toolName: "read_file", ... }
  前端：创建 toolcall 消息（timestamp: T2）
       → 排序后在 thinking 和 assistant 之间

T3 (1712000001500)
  ↓
  后端发送：tool_call_update { status: "completed" }
  前端：更新 toolcall 状态

T4 (1712000002000)
  ↓
  后端发送：agent_message_chunk { content: "我已经读取了..." }
  前端：检测到 timestamp 变化（>100ms），创建新的 assistant 消息段
       → 排序后在 toolcall 后面

T5 (1712000002500)
  ↓
  后端发送：agent_message_chunk { content: "文件内容是..." }
  前端：追加到当前 assistant 消息段

T6 (1712000003000)
  ↓
  后端发送：streamEnd
  前端：结束流式，auto-scroll
```

### 最终渲染顺序

```
┌─────────────────────────────────────────┐
│  [Thinking] 让我想想这个问题...          │  ← timestamp: T0-1
├─────────────────────────────────────────┤
│  [ToolCall] 读取文件 ✓ (120ms)          │  ← timestamp: T2
├─────────────────────────────────────────┤
│  [Assistant] 我已经读取了文件内容...     │  ← timestamp: T4
│              文件内容是...               │  ← timestamp: T5
└─────────────────────────────────────────┘
```

## 后端实现建议（Node.js）

```javascript
// 发送 thinking
function sendThinking(res, content, baseTimestamp) {
  sendSSE(res, 'agent_thought_chunk', {
    content,
    timestamp: baseTimestamp - 1,  // 比 assistant 早 1ms
  });
}

// 发送 toolcall
function sendToolCall(res, toolCallData, timestamp) {
  sendSSE(res, 'tool_call', {
    ...toolCallData,
    timestamp: timestamp || Date.now(),
  });
}

// 发送 toolcall 更新
function sendToolCallUpdate(res, toolCallId, status, detail, timestamp) {
  sendSSE(res, 'tool_call_update', {
    toolCallId,
    status,
    detail,
    timestamp: timestamp || Date.now(),
  });
}

// 发送 assistant 回复（支持多段）
function sendAssistantChunk(res, content, timestamp) {
  sendSSE(res, 'agent_message_chunk', {
    content,
    timestamp: timestamp || Date.now(),
  });
}

// 完整流程示例
async function handleChat(res, query) {
  const baseTimestamp = Date.now();
  
  // 1. 发送 streamStart
  sendSSE(res, 'streamStart', { timestamp: baseTimestamp });
  
  // 2. 发送 thinking
  sendThinking(res, '让我分析一下这个问题...', baseTimestamp);
  
  // 3. 发送 toolcall
  const toolcallTimestamp = baseTimestamp + 1000;
  sendToolCall(res, {
    toolCallId: 'tc_001',
    toolName: 'read_file',
    title: '正在读取文件',
    content: '/path/to/file.txt',
    status: 'running',
  }, toolcallTimestamp);
  
  // 4. 模拟工具执行
  await executeTool();
  
  // 5. 发送 toolcall 更新
  sendToolCallUpdate(res, 'tc_001', 'completed', '完成 (120ms)', toolcallTimestamp + 120);
  
  // 6. 发送 assistant 回复（toolcall 后）
  const assistantTimestamp = toolcallTimestamp + 2000;
  sendAssistantChunk(res, '我已经读取了文件内容...', assistantTimestamp);
  sendAssistantChunk(res, '文件内容如下...', assistantTimestamp + 100);
  
  // 7. 发送 streamEnd
  sendSSE(res, 'streamEnd', { timestamp: Date.now() });
}
```

## 前端处理逻辑

### chatStore.js 关键代码

```javascript
appendAssistantChunk: (chunk, options) => {
  const { assistantMessageId, lastAssistantTimestamp } = get();
  const timestamp = options?.timestamp ?? Date.now();

  // 如果是新的一段（时间戳差异 > 100ms），创建新的 assistant 消息
  if (options?.timestamp && lastAssistantTimestamp && 
      Math.abs(timestamp - lastAssistantTimestamp) > 100) {
    const id = uid();
    set((state) => ({
      assistantMessageId: id,
      lastAssistantTimestamp: timestamp,
      messages: [
        ...state.messages,
        { id, role: "assistant", content: chunk, timestamp },
      ],
    }));
    return;
  }

  // 否则追加到当前 assistant 消息
  set((state) => ({
    messages: state.messages.map((message) =>
      message.id === assistantMessageId
        ? { ...message, content: `${message.content}${chunk}` }
        : message,
    ),
  }));
},
```

### getAllMessages - 按时间戳排序

```javascript
getAllMessages: () => {
  const { messages, toolCalls } = get();
  
  // 将 toolCalls 转换为消息格式
  const toolCallMessages = toolCalls.map((tool) => ({
    id: `tool_${tool.id}`,
    role: "tool_call",
    toolCall: tool,
    timestamp: tool.timestamp ?? Date.now(),
  }));

  // 合并并按时间戳排序
  return [...messages, ...toolCallMessages].sort(
    (a, b) => (a.timestamp ?? 0) - (b.timestamp ?? 0)
  );
},
```

## 常见问题

### Q1: 为什么 thinking 总是在 assistant 前面？

**答**: Thinking 的时间戳 = `assistantTimestamp - 1`，排序时自然在前面。

### Q2: 如何在 toolcall 后继续输出 text？

**答**: 发送 `agent_message_chunk` 时使用**新的时间戳**（比 toolcall 晚），前端会检测到时间戳变化并创建新的 assistant 消息段。

### Q3: 多个 toolcall 如何排序？

**答**: 每个 toolcall 使用不同的 timestamp，按时间戳自然排序。

### Q4: 时间戳差异多少会创建新的消息段？

**答**: 默认 > 100ms 会创建新段，可在 `appendAssistantChunk` 中调整阈值。

---

## 总结

| 消息类型 | 时间戳策略 | 目的 |
|---------|-----------|------|
| Thinking | `assistantTimestamp - 1` | 确保在 assistant 前面 |
| ToolCall | `Date.now()`（发送时） | 按实际发生时间排序 |
| Assistant（第一段） | `baseTimestamp` | 初始回复 |
| Assistant（后续段） | `toolcallTimestamp + Δ` | 在 toolcall 后继续输出 |

**核心**: 所有消息通过**时间戳排序**实现交错显示，后端控制时间戳即可控制显示顺序。
