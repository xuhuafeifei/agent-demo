# 工具安全性重构技术方案

## 📋 1. 项目背景与现状

### 1.1 项目概述

- **项目名称**：agent-demo
- **项目性质**：个人日常使用的非专业编程 AI 助手
- **技术栈**：Node.js + TypeScript + @mariozechner/pi-agent-core
- **使用场景**：Web 界面 + QQ 机器人，支持多通道对话

### 1.2 当前安全现状

**现有安全机制**：
- 基础路径守卫：防止路径逃逸（`../` 相对路径）
- 工具注册表：通过配置文件控制工具可用性
- 简单工具分类：`tools`、`customTools`、`innerTools`

**存在的主要问题**：
1. `shellExecute` 工具完全开放，可执行任意命令
2. 工具权限粒度粗糙，无法精细控制
3. 缺乏文件类型限制，可能访问二进制文件
4. 没有危险路径黑名单
5. 缺乏执行确认机制
6. 没有资源限制和超时控制

### 1.3 设计目标

**核心原则**：
- 简单易用：用户配置直观，不需要复杂的安全知识
- 渐进式增强：从简单到复杂，逐步完善安全机制
- 实用优先：满足个人使用场景，不过度设计
- 性能友好：不影响用户体验和响应速度

**安全目标**：
- 防止 AI 误操作导致的系统损害
- 保护敏感文件和配置不被泄露
- 限制资源使用，防止系统资源耗尽
- 提供足够的灵活性，支持不同使用场景

#### 2.2.2 路径安全检查机制

**核心功能**：
1. 拒绝相对路径逃逸（`../`）
2. 解析符号链接，获取真实路径
3. 应用全局黑名单检查
4. 应用用户自定义黑名单检查
5. 检查文件类型（文本文件限制）

**实现思路**：
```typescript
interface PathCheckResult {
  allowed: boolean;
  realPath: string;
  reason?: string;
}

async function checkPathSafety(
  inputPath: string,
  config: SecurityConfig
): Promise<PathCheckResult> {
  // 1. 检查相对路径
  if (inputPath.includes('..')) {
    return { allowed: false, realPath: '', reason: '相对路径不被允许' };
  }

  // 2. 解析真实路径（处理符号链接）
  const realPath = await fs.realpath(inputPath);

  // 3. 检查全局黑名单
  if (matchesGlobalDenyList(realPath)) {
    return { allowed: false, realPath, reason: '路径在全局黑名单中' };
  }

  // 4. 检查用户自定义黑名单
  if (matchesUserDenyList(realPath, config.denyPaths)) {
    return { allowed: false, realPath, reason: '路径在用户黑名单中' };
  }

  // 5. 检查文件类型
  if (await isBinaryFile(realPath)) {
    return { allowed: false, realPath, reason: '只允许访问文本文件' };
  }

  return { allowed: true, realPath };
}
```

#### 2.2.3 文件类型检测机制

**混合策略**：
1. 扩展名快速判断（常见文本/二进制文件）
2. 无扩展名时检测文件头 magic number
3. 保守策略：默认允许未知文件

```typescript
async function isTextFile(path: string): Promise<boolean> {
  // 1. 扩展名快速判断
  const ext = path.extname(path).toLowerCase();
  if (TEXT_EXTENSIONS.has(ext)) return true;
  if (BINARY_EXTENSIONS.has(ext)) return false;

  // 2. 无扩展名时检测文件头
  const buffer = await readFileHeader(path, 8);
  if (isBinaryMagicNumber(buffer)) return false;

  // 3. 保守策略：默认允许
  return true;
}

const TEXT_EXTENSIONS = new Set([
  '.txt', '.md', '.json', '.yaml', '.yml', '.xml',
  '.js', '.ts', '.py', '.sh', '.bash', '.zsh',
  '.html', '.css', '.scss', '.less',
  '.c', '.cpp', '.h', '.hpp', '.java', '.go', '.rs',
]);

const BINARY_EXTENSIONS = new Set([
  '.exe', '.dll', '.so', '.dylib', '.a', '.lib',
  '.png', '.jpg', '.jpeg', '.gif', '.bmp', '.ico',
  '.mp3', '.mp4', '.avi', '.mov', '.wav',
  '.zip', '.tar', '.gz', '.bz2', '.7z', '.rar',
  '.pdf', '.doc', '.docx', '.xls', '.xlsx',
]);
```

### 2.3 受限宿主机沙箱方案

#### 2.3.1 设计理念

**核心思想**：
- 在宿主机上执行命令，但通过各种限制手段约束进程
- 平衡安全性和性能，适合个人使用场景
- 避免容器/虚拟机的复杂性和性能开销

**安全边界**：
```
┌─────────────────────────────────────┐
│         宿主机操作系统          │
│  ┌───────────────────────────┐   │
│  │     受限进程            │     │
│  │  - chroot 文件系统隔离 │   │
│  │  - seccomp 系统调用过滤│   │
│  │  - ulimit 资源限制     │   │
│  │  - 网络命名空间隔离   │   │
│  └───────────────────────────┘   │
└─────────────────────────────────────┘
```

#### 2.3.2 技术手段

**文件系统隔离**：
- `chroot`：限制文件系统访问范围
- `realpath`：解析符号链接，防止路径绕过
- 只读挂载：防止意外修改系统文件

**系统调用过滤**：
- `seccomp`：过滤危险系统调用
- 禁止网络相关系统调用（如果需要）
- 禁止文件系统修改系统调用（如果需要）

**资源限制**：
- `ulimit`：限制文件描述符、进程数等
- `cgroups`：限制 CPU、内存使用
- 执行超时：防止无限循环

**网络隔离**：
- 网络命名空间：完全隔离网络栈
- iptables 规则：限制网络访问

#### 2.3.3 实现策略

**阶段 1：基础限制（快速实现）**
```typescript
async function basicSandbox(command: string, config: SandboxConfig): Promise<SandboxResult> {
  // 1. 命令预检查
  await preExecuteCheck(command);

  // 2. 启动受限子进程
  const child = spawn(command, {
    cwd: config.workspace,
    timeout: config.timeout || 30000,
    env: sanitizeEnvironment(process.env),
  });

  // 3. 收集输出和等待完成
  const result = await waitForCompletion(child);

  // 4. 执行后清理
  await postExecuteCleanup(result);

  return result;
}
```

**阶段 2：增强隔离（可选）**
```typescript
async function enhancedSandbox(command: string, config: SandboxConfig): Promise<SandboxResult> {
  // 1. 创建临时沙箱目录
  const sandboxDir = await createSandboxDirectory();

  // 2. 挂载必要的系统文件
  await mountSystemFiles(sandboxDir);

  // 3. 挂载工作目录
  await mountWorkspace(sandboxDir, config.workspace, config.readOnly);

  // 4. 使用 unshare 创建隔离环境
  const result = await executeInSandbox(command, sandboxDir, config);

  // 5. 清理沙箱
  await cleanupSandbox(sandboxDir);

  return result;
}
```

#### 2.3.4 命令安全检查

**多层检查机制**：

```typescript
async function preExecuteCheck(command: string): Promise<void> {
  // 1. 命令长度限制
  if (command.length > 10000) {
    throw new Error('命令过长');
  }

  // 2. 危险命令模式检测
  const dangerousPatterns = [
    /rm\s+-rf/i,           // 强制删除
    /dd\s+if=/i,          // 磁盘操作
    />\s*\/dev\/(sd[a-z]|null)/i,  // 设备操作
    /curl.*\|.*sh/i,      // 命令注入
    /mkfs/i,              // 文件系统创建
  ];

  for (const pattern of dangerousPatterns) {
    if (pattern.test(command)) {
      throw new Error('检测到危险命令模式');
    }
  }

  // 3. 路径参数检查
  const paths = extractPaths(command);
  for (const path of paths) {
    const check = await checkPathSafety(path);
    if (!check.allowed) {
      throw new Error(`路径访问被拒绝: ${check.reason}`);
    }
  }

  // 4. 网络访问检查
  if (!config.network && hasNetworkAccess(command)) {
    throw new Error('网络访问被禁用');
  }
}
```
